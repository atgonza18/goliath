Goliath Email Integration — Complete Setup Guide

This guide sets up two-way email so Goliath can send and receive emails on your behalf through your Office 365 account.

🏗️ How It Works

OUTBOUND (Goliath sends email as you):
You tell Goliath → drafts email → you approve in Telegram
→ Goliath drops email in Gmail with tag [SEND: recipient@email.com]
→ Power Automate picks it up → sends via your Office 365
→ Recipient gets email FROM your work address

INBOUND (Goliath processes your incoming email):
Email arrives in your Office 365 inbox
→ Power Automate forwards it to Goliath's Gmail with tag [INBOX: sender]
→ Goliath picks it up, drafts a response
→ You approve/edit/reject in Telegram
→ If approved, goes through the outbound flow above

🐍 PART 3: Goliath Code Snippets

These go in the Goliath codebase. No additional Python packages needed — smtplib, imaplib, and email are all built into Python.

A. Email Sender (email_service.py)

import smtplib
import logging
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart
import os

logger = logging.getLogger(__name__)

GMAIL = os.getenv("GOLIATH_GMAIL")
GMAIL_PWD = os.getenv("GOLIATH_GMAIL_APP_PASSWORD")

def send_outbound_email(to: str, subject: str, body: str, html: bool = False):
    # Send email via Gmail SMTP for Power Automate to relay via Office 365.
    # Subject is tagged so PA knows where to route it.
    msg = MIMEMultipart()
    msg['From'] = GMAIL
    msg['To'] = GMAIL  # Send to ourselves — PA picks it up
    msg['Subject'] = f"[SEND: {to}] {subject}"

    content_type = 'html' if html else 'plain'
    msg.attach(MIMEText(body, content_type))

    with smtplib.SMTP_SSL('smtp.gmail.com', 465) as server:
        server.login(GMAIL, GMAIL_PWD)
        server.sendmail(GMAIL, [GMAIL], msg.as_string())

    logger.info(f"Outbound relay email sent to={to} subject={subject!r}")

B. Email Poller (email_poller.py)

import imaplib
import email
import re
import logging
import os
from email.header import decode_header

logger = logging.getLogger(__name__)

GMAIL = os.getenv("GOLIATH_GMAIL")
GMAIL_PWD = os.getenv("GOLIATH_GMAIL_APP_PASSWORD")

def get_body(msg):
    # Extract plain text body from email message.
    if msg.is_multipart():
        for part in msg.walk():
            if part.get_content_type() == 'text/plain':
                return part.get_payload(decode=True).decode('utf-8', errors='replace')
    return msg.get_payload(decode=True).decode('utf-8', errors='replace')

def poll_inbox():
    # Check Gmail for [INBOX:] tagged emails from Power Automate.
    mail = imaplib.IMAP4_SSL('imap.gmail.com')
    mail.login(GMAIL, GMAIL_PWD)
    mail.select('INBOX')

    _, msg_ids = mail.search(None, 'SUBJECT "[INBOX:"')
    messages = []

    for msg_id in msg_ids[0].split():
        _, data = mail.fetch(msg_id, '(RFC822)')
        msg = email.message_from_bytes(data[0][1])
        subject = str(msg['Subject'])

        match = re.match(r'\[INBOX:\s*(.+?)\]\s*(.*)', subject)
        if match:
            messages.append({
                'from': match.group(1),
                'subject': match.group(2),
                'body': get_body(msg),
                'message_id': msg['Message-ID']
            })
            # Mark for deletion after processing
            mail.store(msg_id, '+FLAGS', '\\Deleted')

    mail.expunge()
    mail.logout()
    logger.info(f"Polled inbox: {len(messages)} new messages")
    return messages

🔧 Troubleshooting

  • Flow not triggering: Gmail triggers can take 1-5 min. Check "Run history" in Power Automate. If no runs, re-authenticate the Gmail connection.
  • Expression errors: Make sure you paste expressions in the Expression tab (fx icon), NOT in the text field directly.
  • "Invalid connection" error: Re-authenticate both Gmail and Office 365 connectors under Connections in Power Automate.
  • Email not deleting from Gmail: The Gmail connector needs "modify" permission. Re-authorize if needed.
  • Loop protection: Flow #2 has a condition to skip emails from nimrodgreatski to prevent infinite loops. Make sure this condition is set correctly.
